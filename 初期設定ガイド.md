# ぺろんちょOS v0.6 初期設定ガイド

この更新では、これまでのタスクを残したまま、クラウド保存とiPhone通知を追加します。

## 1. Supabaseを準備する

1. `https://supabase.com/` で無料アカウントを作る
2. 「New project」でプロジェクトを作る
3. 左側の「SQL Editor」を開く
4. このフォルダの `supabase/schema.sql` を開き、中身を貼り付けて実行する
5. 「Project Settings」→「API」で次の3つを確認する
   - Project URL
   - anon public key
   - service_role key

`service_role key` は秘密情報です。GitHubやチャットへ貼らないでください。

## 2. 通知用の鍵を作る

パソコンでこのフォルダを開き、ターミナルから次を実行します。

```bash
npx web-push generate-vapid-keys
```

表示されたPublic KeyとPrivate Keyを控えます。Private Keyは秘密情報です。

## 3. Vercelへ環境変数を登録する

Vercelの対象プロジェクトで「Settings」→「Environment Variables」を開き、次を登録します。

| 名前 | 入れる内容 |
| --- | --- |
| `NEXT_PUBLIC_SUPABASE_URL` | SupabaseのProject URL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Supabaseのanon public key |
| `SUPABASE_SERVICE_ROLE_KEY` | Supabaseのservice_role key |
| `NEXT_PUBLIC_VAPID_PUBLIC_KEY` | VAPIDのPublic Key |
| `VAPID_PRIVATE_KEY` | VAPIDのPrivate Key |
| `VAPID_SUBJECT` | `mailto:`＋自分のメールアドレス |
| `CRON_SECRET` | 自分で作った長いランダム文字列 |

以前登録した `OPENAI_API_KEY` は、そのまま残してください。

## 4. GitHubへ更新版をアップロードする

1. ZIPをすべて展開する
2. GitHubの `peloncho1011/peroncho` を開く
3. 「Add file」→「Upload files」
4. 展開したフォルダの中身をすべてアップロード
5. 「Commit changes」
6. VercelがReadyになるまで待つ

## 5. iPhoneで通知を有効にする

1. SafariでぺろんちょOSを開く
2. 共有ボタンから「ホーム画面に追加」
3. ホーム画面のぺろんちょOSを開く
4. 設定→アカウントでメールアドレスを入力
5. 届いたメールのログインリンクを押す
6. 設定→お知らせ→「お知らせを有効にする」
7. iPhoneの確認画面で「許可」
8. 「テスト通知を送る」で確認する

通知は、期限3日前・前日・当日の朝8時に届きます。定例タスクに設定された通知日も引き継がれます。

## データの引き継ぎ

初めてログインしたとき、現在のiPhone内にあるタスクをクラウドへ保存します。念のため、更新前に設定画面の「バックアップを書き出す」も実行してください。
